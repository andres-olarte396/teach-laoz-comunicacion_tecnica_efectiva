# PENSUM DE COMPETENCIAS: COMUNICACIÓN TÉCNICA EFECTIVA

## PERFIL DE EGRESO

Al finalizar este curso, el estudiante será capaz de:

1. Producir documentación técnica clara, estructurada y mantenible.
2. Argumentar decisiones de arquitectura mediante documentos estandarizados (ADRs/RFCs).
3. Gestionar la comunicación en situaciones críticas sin generar deuda técnica social.

## MATRIZ DE COMPETENCIAS POR MÓDULO

| Módulo | Competencia Específica (Saber Hacer) | Resultado de Aprendizaje (Evidencia) |
| :--- | :--- | :--- |
| **Módulo 0** | Configurar herramientas de linting de prosa. | Pipeline de CI fallando por errores gramaticales (Vale). |
| **Módulo 1** | Aplicar principios de claridad y concisión. | Texto técnico reescrito reduciendo el word-count 30%. |
| **Módulo 2** | Documentar codebase para mantenibilidad. | PR con README estructurado y Docstrings significativos. |
| **Módulo 3** | Proponer cambios complejos asíncronamente. | Design Doc (RFC) aprobado por el equipo. |
| **Módulo 4** | Realizar análisis de incidentes constructivos. | Reporte Postmortem identificando causa raíz sistémica. |
| **Módulo 5** | Integrar capacidades de comunicación técnica. | Portal de desarrollador funcional (Mockup). |
