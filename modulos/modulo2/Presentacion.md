# Presentación Módulo 2: Documentación técnica clara y útil

## Objetivo

Desarrollar habilidades para crear documentación técnica comprensible, estructurada y útil para distintos públicos en proyectos de software.

## Temas clave

- Estructura básica de documentación técnica (README, manuales, API docs)
- Buenas prácticas de redacción y formato
- Uso de ejemplos de código y diagramas
- Adaptación de la documentación a distintos perfiles (desarrollador, usuario final, soporte)

## Metodología

- Análisis de ejemplos reales de documentación
- Ejercicios de redacción y revisión entre pares
- Uso de herramientas de documentación automatizada

## Recursos

- Repositorios con documentación ejemplar
- Plantillas de README y API docs
- Videos sobre documentación efectiva

## Evaluación

- Revisión de documentación creada por los participantes
- Checklist de buenas prácticas
